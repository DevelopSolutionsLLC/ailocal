# CLAUDE.md — ailocal repo primer

Agent primer for this repository. Keep it lean: every edit here should stay
under ~50 lines. Do not add a new doc file without merging or deleting an old one.

## What this repo is

Tooling to run AI coding clients (Claude Code, Codex CLI, VS Code Copilot Chat)
against **local** models on Apple Silicon — no cloud, no code changes to the tools.

Architecture: **Ollama** runs models natively (Metal/MLX GPU). **LiteLLM** (one
Docker container, `127.0.0.1:4000`) sits in front as an OpenAI+Anthropic-compatible
proxy and exposes **role names** (`coder-main`, `coder-agent`, `coder-fast`,
`deep-think`, `deep-think-more`, `supervisor`, `embed`) instead of raw model tags.
Clients point at `localhost:4000`. Claude/OpenAI names (`claude-*`, `gpt-*`) are
aliased onto roles via `router_settings.model_group_alias`.

## Golden rule

**Use role names only** in client configs and scripts — never backend model tags
(`qwen3-coder:30b`, etc.). Roles decouple configs from the models behind them.

Reasoning: only `deep-think`/`deep-think-more` think (DeepSeek-R1; served with
their reasoning stream **merged** into the answer text via
`merge_reasoning_content_in_choices`, so VS Code shows visible `<think>` instead of
a silent spinner). `coder-*`/`supervisor` are pinned to `reasoning_effort: "none"`
(invisible reasoning reads as a hang in OpenAI-format clients). Personas are
injected by the `persona_injector` hook from `config/personas/*.md`; reasoners get
none (DeepSeek: no system prompt) and run at temp 0.6.

## Where things live

- `config/models.yaml` — role → backend mapping (generated by `install.sh`).
- `config/litellm/config.yaml` — proxy config; the `model_list` block is **generated**
  between the GENERATED markers. `config/litellm/persona_injector.py` — the pre-call
  hook that injects each role's persona into the client's system message.
- `config/personas/` — `_core.md` (shared grounded directives) + per-role enhancers;
  the single source read by the hook. Reasoners get none (DeepSeek: no system prompt).
- `config/clients/` — `configure.zsh`/`finalize.zsh` (sourced via `.zshrc` markers),
  per-client dirs `claude/`, `codex/`, `copilot/`, `continue/`, and `model_catalog.json`,
  deployed to `~/.config/ailocal/` (XDG; never touches `~/.claude` or `~/.codex`).
- `scripts/` — `install.sh`, `install-clients.sh`, `sync-models.sh` → `sync-models.py`,
  `start/stop/update/teardown`, `doctor.sh`, `smoke-test.sh`.

## Generated files — never hand-edit

`sync-models.py` regenerates from `models.yaml`: the `model_list` block of
`config/litellm/config.yaml` (backend, num_ctx, sampling, capability flags — between
the GENERATED markers) and `config/clients/model_catalog.json`. After changing a model:
edit `config/models.yaml` → `./scripts/sync-models.sh` → `./scripts/start.sh` (restart
also reloads persona `.md` edits into the hook).

## Verify

`./scripts/doctor.sh` (0=healthy, 2=degraded) and `./scripts/smoke-test.sh`.
`bash -n` any edited script. `./scripts/sync-models.sh` must produce zero diff.

## Conventions

Shell: `set -euo pipefail`, reuse the `info/warn/step/backup` helpers. Python: stdlib
only. Never commit `.env` or secrets; ports bind `127.0.0.1` only. No Claude commit
attribution — Victor's identity only. Never `git push` without explicit approval.
